# 結論 — 完整整合

## 六層的力量

我們已經探索了代理 harness 工程的六個核心層級：

1. **🔁 迴圈**：觀察-決策-行動-驗證的核心迴圈
2. **🛠️ 工具**：讓代理在世界中行動的能力
3. **🧠 上下文**：管理代理的工作記憶
4. **💾 持久化**：跨會話的記憶能力
5. **✅ 驗證**：確保代理行為正確的檢查機制
6. **🛑 約束**：定義代理不能做什麼的信任基礎

## bootstrap_session() — 全部連接在一起

claw-code 的 `PortRuntime.bootstrap_session()` 是所有層級如何組合的最佳範例：

1. 建構工作區上下文（第 3 層）
2. 運行設置和預取（第 3 層）
3. 載入工具和命令（第 2 層）
4. 路由提示到相關模組（第 1 層）
5. 建構執行注冊表（第 2 層）
6. 推斷權限拒絕（第 6 層）
7. 提交訊息到查詢引擎（第 1 層）
8. 持久化會話（第 4 層）
9. 記錄歷史事件（第 4 層）

每一步都涉及至少一個層級。這不是線性的——它們相互交織。

## 關鍵洞察

> Harness 不是樣板程式碼——它是核心工程。

Claude Code 原始碼洩漏揭示的最重要的事實是：約 40% 的能力來自 harness 而非模型。這意味著：

- 更好的工具過濾 = 更好的代理表現
- 更好的上下文管理 = 更少的幻覺
- 更好的約束設計 = 更安全的部署
- 更好的持久化 = 更連貫的體驗

## 如何進一步探索

- 克隆 [claw-code](https://github.com/instructkr/claw-code) 並閱讀完整原始碼
- 探索 Rust 層以了解生產級實作模式
- 運行 `examples/07_full_harness.py` 看六層如何協同工作
- 嘗試修改範例：改變權限、添加新工具、調整壓縮策略

## Agent Harness 是造就差異的 40%

模型很強大，但 harness 讓它們變得**有用**。這就是 harness 工程的核心訊息。
