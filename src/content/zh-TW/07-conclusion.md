## 結論：完整整合

我們已經探索了代理 harness 工程的六個核心層級。現在讓我們看看它們如何組合成一個完整的系統。

### 六層總結

- **迴圈** — 觀察-決策-行動-驗證的核心循環，代理的心跳
- **工具** — 讓代理在世界中行動的能力，從註冊到分派
- **上下文** — 管理代理的工作記憶，包括預取和壓縮
- **持久化** — 跨會話的記憶能力，讓長期任務成為可能
- **驗證** — 確保代理行為正確的檢查機制，從成本追蹤到鉤子系統
- **約束** — 定義代理不能做什麼的信任基礎，權限引擎和沙箱

每一層都解決一個獨立的問題，但它們並非孤立運作。

### bootstrap_session()：所有層級的交會點

claw-code 的 `PortRuntime.bootstrap_session()` 是理解六層如何組合的最佳入口。這個函式在一次會話啟動時：

- 建構工作區上下文（上下文層）
- 運行設置和預取（上下文層）
- 載入工具和命令（工具層）
- 路由提示到相關模組（迴圈層）
- 建構執行註冊表（工具層）
- 推斷權限拒絕（約束層）
- 提交訊息到查詢引擎（迴圈層）
- 持久化會話（持久化層）
- 記錄歷史事件（持久化層）

每一步都涉及至少一個層級。它們相互交織，而不是線性排列。

### 關鍵洞察

> Harness 不是樣板程式碼——它是核心工程。

Claude Code 大約 40% 的能力來自 harness 而非模型。這意味著：

- 更好的工具過濾 = 更好的代理表現
- 更好的上下文管理 = 更少的幻覺
- 更好的約束設計 = 更安全的部署
- 更好的持久化 = 更連貫的體驗

Harness 工程不是附加功能，而是決定代理品質的關鍵因素。

### 如何進一步探索

- 閱讀 claw-code 的完整原始碼，逐層對照本指南的概念
- 探索 Rust 層以了解生產級實作模式
- 運行完整 harness 範例，觀察六層如何協同工作
- 嘗試修改：改變權限設定、添加新工具、調整壓縮策略
- 設計你自己的 harness，從最簡單的迴圈開始逐步擴展

### Agent Harness 是造就差異的 40%

模型很強大，但 harness 讓它們變得**有用**。模型提供智慧，harness 提供結構、記憶、安全性和可靠性。理解並掌握 harness 工程，就是理解如何將一個語言模型轉變為一個真正可靠的代理。

這就是 Agent Harness 工程的核心訊息。現在，去建造你自己的 harness 吧。
