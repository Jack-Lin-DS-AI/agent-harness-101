## 什麼是 Agent Harness 工程？

當你使用像 Claude Code 這樣的 AI 編程代理時，你可能會以為所有的魔法都在模型裡。事實並非如此。大約 40% 讓 Claude Code 如此有效的能力來自 **harness**——將語言模型轉變為可靠代理的周圍工程。

這就是 **Agent Harness 工程**：建構讓 LLM 在真實世界中發揮作用的腳手架的學科。

## 核心公式

> **Agent = 模型 + Harness**

這個框架來自 Mitchell Hashimoto 的觀察：Claude Code 中的非模型程式碼不是樣板——它是產品能力的核心。模型提供智慧，Harness 提供結構、記憶、安全性和可靠性。

## 六層架構概述

每個代理 harness 都可以透過六個層次來理解，每個層次解決一個獨立的問題：

- **迴圈（Loop）** — 驅動代理前進的觀察-決策-行動-驗證循環
- **工具（Tools）** — 代理在世界中行動的方式：註冊、過濾、分派
- **上下文（Context）** — 代理的工作記憶：系統提示、預取、壓縮
- **持久化（Persistence）** — 跨會話記憶：儲存與恢復狀態
- **驗證（Verification）** — 可觀察性與安全性：成本追蹤、鉤子、自我審計
- **約束（Constraints）** — 代理不能做什麼：權限、信任模式、沙箱

這些層次相互組合。每一層本身都很簡單，但結合在一起，它們便能產生一個能力強大且值得信賴的代理。

## claw-code 如何幫助我們學習

**claw-code** 是 Claude Code 核心架構的 Python 和 Rust 重新實作。它的存在是為了讓這些模式變得可見且可研究。

原始的 Claude Code 是一個生產級的 TypeScript 程式碼庫——龐大、經過最佳化，有時候不太適合教學閱讀。claw-code 使用 Python（追求清晰）和 Rust（追求生產級模式）重新實作相同的概念，提供一個更小的程式碼庫，讓每個設計決策都清晰可讀。

在本指南中，我們會引用 claw-code 中的具體檔案和函式。當你看到像 `run_turn_loop()` 這樣的函式名稱或像 `context.py` 這樣的檔案時，那是你可以閱讀和修改的真實程式碼。

## 你將學到什麼

讀完本指南後，你將理解：

- 為什麼 harness 與模型同等重要
- 每一層如何獨立運作以及如何協同組合
- 在每個代理框架中出現的通用模式
- 如何閱讀和擴展 claw-code 來實驗你自己的 harness 設計

讓我們從最基礎的層次開始：迴圈。
